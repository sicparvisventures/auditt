# AUDITFLOW - COMPREHENSIVE LOVABLE PROMPT
## Ultra Gedetailleerde Specificatie voor Identieke Recreatie

---

## OVERZICHT
Maak een identieke kopie van de bestaande AuditFlow SaaS audit platform. Dit is een volledig functionerende multi-tenant SaaS applicatie voor interne audits met professionele branding, onboarding, en complete audit functionaliteit.

---

## TECHNISCHE STACK
- **Frontend**: Next.js 14 met App Router, React 18, TypeScript
- **Styling**: Tailwind CSS met custom kleurenschema
- **Backend**: Supabase (PostgreSQL database, Auth, Storage)
- **State Management**: React Context (AuthProvider)
- **Forms**: React Hook Form met Zod validation
- **Charts**: Recharts voor KPI visualisatie
- **PDF**: jsPDF + html2canvas voor rapporten
- **Icons**: Lucide React
- **Deployment**: Cloudflare Pages ready

## AUDIT TOOL WERKING

### Nieuwe Audit Flow
1. **Filiaal Selectie**: Grid van beschikbare filialen
2. **Audit Form**: Checklist items per categorie
3. **Scoring**: Automatische berekening
4. **Photo Upload**: Per checklist item
5. **Actions**: Automatische actie creatie

### Audit Form Structuur
- **Header**: Filiaal info, manager, datum, voortgang bar
- **Categorieën**: Groepeerde checklist items
- **Per Item**:
  - Titel en beschrijving
  - Gewicht indicator
  - OK/Niet OK knoppen
  - Opmerkingen textarea
  - Verbeterpunt (alleen bij Niet OK)
  - Photo upload component
- **Algemene Opmerkingen**: Textarea aan het einde
- **Submit**: "Audit Voltooien" knop

### Checklist Items (Restaurant Template)
**Categorieën**:
1. **Exterieur & Toegankelijkheid**
   - Gevel en ramen schoon
   - Parkeerplaats netjes
   - Toegankelijkheid voor mindervaliden

2. **Interieur & Hygiëne**
   - Vloeren schoon en droog
   - Tafels en stoelen netjes
   - Sanitaire voorzieningen schoon

3. **Keuken & Voedselveiligheid**
   - Koeling op juiste temperatuur
   - HACCP procedures gevolgd
   - Persoonlijke hygiëne medewerkers

4. **Service & Klanttevredenheid**
   - Vriendelijke bediening
   - Wachtrijen beheerd
   - Klachtenprocedure bekend

5. **Veiligheid & Procedures**
   - Brandveiligheid in orde
   - Eerste hulp materiaal aanwezig
   - Procedures bekend bij medewerkers

### Scoring Systeem
- **Scale**: 0-5 punten per item
- **Weight**: Elk item heeft gewicht (standaard 1.0)
- **Calculation**: Gewogen gemiddelde
- **Pass Threshold**: 80% (4.0/5.0)
- **Breakdown**: Per categorie en totaal

### Photo Upload
- **Component**: Drag & drop interface
- **Formats**: JPG, PNG (max 10MB)
- **Limit**: 5 foto's per item
- **Storage**: Supabase Storage
- **Preview**: Thumbnail weergave

### Action Creation
- **Automatic**: Bij "Niet OK" resultaten
- **Fields**: Titel, beschrijving, urgentie, deadline
- **Assignment**: Toegewezen aan filiaal manager
- **Status**: Pending → In Progress → Completed → Verified

## DATABASE SCHEMA

### Core Tables
```sql
-- Organizations (Multi-tenant)
organizations (
  id UUID PRIMARY KEY,
  name VARCHAR(255),
  slug VARCHAR(100) UNIQUE,
  tier VARCHAR(50),
  status VARCHAR(50),
  primary_color VARCHAR(7),
  secondary_color VARCHAR(7),
  accent_color VARCHAR(7),
  background_color VARCHAR(7),
  text_color VARCHAR(7),
  primary_font VARCHAR(100),
  accent_font VARCHAR(100),
  logo_url TEXT,
  max_users INTEGER,
  max_filialen INTEGER,
  max_audits_per_month INTEGER
)

-- Users
gebruikers (
  id UUID PRIMARY KEY,
  email CITEXT UNIQUE,
  naam VARCHAR(255),
  rol user_role,
  telefoon VARCHAR(20),
  actief BOOLEAN,
  organization_id UUID REFERENCES organizations(id)
)

-- Locations
filialen (
  id UUID PRIMARY KEY,
  naam VARCHAR(255),
  locatie VARCHAR(255),
  district_manager_id UUID REFERENCES gebruikers(id),
  adres TEXT,
  telefoon VARCHAR(20),
  email CITEXT,
  status filiaal_status,
  organization_id UUID REFERENCES organizations(id)
)

-- Checklist Items
audit_checklist_items (
  id UUID PRIMARY KEY,
  categorie VARCHAR(100),
  titel VARCHAR(255),
  beschrijving TEXT,
  gewicht NUMERIC(3,2),
  volgorde INTEGER,
  actief BOOLEAN,
  organization_id UUID REFERENCES organizations(id)
)

-- Audits
audits (
  id UUID PRIMARY KEY,
  filiaal_id UUID REFERENCES filialen(id),
  district_manager_id UUID REFERENCES gebruikers(id),
  audit_datum DATE,
  status audit_status,
  totale_score NUMERIC(3,2),
  pass_percentage NUMERIC(5,2),
  opmerkingen TEXT
)

-- Audit Results
audit_resultaten (
  id UUID PRIMARY KEY,
  audit_id UUID REFERENCES audits(id),
  checklist_item_id UUID REFERENCES audit_checklist_items(id),
  resultaat audit_result,
  score INTEGER,
  opmerkingen TEXT,
  foto_urls TEXT[],
  verbeterpunt TEXT
)

-- Actions
acties (
  id UUID PRIMARY KEY,
  audit_id UUID REFERENCES audits(id),
  audit_resultaat_id UUID REFERENCES audit_resultaten(id),
  titel VARCHAR(255),
  beschrijving TEXT,
  urgentie urgency_level,
  status action_status,
  toegewezen_aan UUID REFERENCES gebruikers(id),
  deadline DATE,
  actie_onderneem TEXT,
  foto_urls TEXT[],
  voltooid_door UUID REFERENCES gebruikers(id),
  voltooid_op TIMESTAMPTZ,
  geverifieerd_door UUID REFERENCES gebruikers(id),
  geverifieerd_op TIMESTAMPTZ
)

-- Reports
rapporten (
  id UUID PRIMARY KEY,
  audit_id UUID REFERENCES audits(id),
  rapport_url TEXT,
  verstuurd_naar TEXT[],
  verstuur_datum TIMESTAMPTZ,
  status report_status
)

-- Notifications
notificaties (
  id UUID PRIMARY KEY,
  gebruiker_id UUID REFERENCES gebruikers(id),
  type VARCHAR(50),
  titel VARCHAR(255),
  bericht TEXT,
  actie_id UUID REFERENCES acties(id),
  gelezen BOOLEAN
)
```

### Enums
```sql
user_role: 'admin', 'coo', 'district_manager', 'filiaal_manager', 'inspector', 'storemanager', 'developer'
filiaal_status: 'actief', 'inactief'
audit_status: 'in_progress', 'completed', 'cancelled'
audit_result: 'ok', 'niet_ok'
action_status: 'pending', 'in_progress', 'completed', 'verified'
urgency_level: 'low', 'medium', 'high', 'critical'
report_status: 'pending', 'sent', 'failed'
```

### Database Functions
```sql
-- Organization Management
create_new_organization(org_name, org_slug, org_tier, contact_name, contact_email, ...)
get_active_organizations()
get_organization_by_slug(org_slug)
authenticate_user_for_organization(user_email, org_slug)

-- Audit Management
calculate_audit_score(resultaten, checklist_items)
create_actions_from_audit_results(audit_id)
get_organization_stats(org_id)
```
